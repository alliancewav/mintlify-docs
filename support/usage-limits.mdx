## Overview

BeatPass implements usage limits to ensure fair access, prevent abuse, and maintain platform performance. This page consolidates all per-feature limits in one reference.

---

## Upload Limits

### Track Uploads

| Parameter | Limit |
|-----------|-------|
| Accepted formats | WAV, FLAC (lossless only) |
| Max file size | 100 MB |
| Max duration | 15 minutes |
| Min duration | 30 seconds |
| Files per track | 1 (single mixdown) |
| Uploads per day | No hard limit* |

*Excessive uploads in a short period may trigger rate limiting. Upload at a reasonable pace.

<Note>
  BeatPass only accepts lossless audio formats (WAV, FLAC). Compressed formats like MP3 are not accepted. Each track is a single audio file â€” stem uploads are not supported through the platform.
</Note>

## What Happens at Limits

| Scenario | Behavior |
|----------|----------|
| Rate limit hit | 429 error; retry after cooldown |
| Storage limit hit | Cannot save more items; remove old items first |
| Daily limit hit | Wait until next day (UTC) |
| Account limit hit | Feature disabled until resolved |

### Rate Limit Headers

API responses include rate limit information:

```http
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 45
X-RateLimit-Reset: 1700000000
```

## Abuse Prevention

Attempting to circumvent rate limits may result in:

| Violation | Consequence |
|-----------|-------------|
| Automated scraping | IP ban, account suspension |
| Multi-account abuse | All accounts terminated |
| API abuse | API access revoked |
| Spam behavior | Account restrictions |
