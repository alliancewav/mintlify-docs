## Required vs Helpful Fields

### Required for New Tracks

BeatPass requires these fields before a new beat can be saved:

- **Track Name**
- **Key**
- **Scale**
- **BPM**

Artists defaults to the uploader's profile, and cover art can default from a collection or **AI Cover Presets**, so neither needs to be manually set every time.

### Helpful for Discovery

These are not required, but they improve how easy the beat is to find and understand:

- **Description**
- **Genres**
- **Tags**
- **Collection / Album assignment**
- **Sample-safe status**

---

## Why Metadata Matters

Good metadata helps BeatPass do a better job of:

- Showing the right beats in search and browse results
- Matching buyers to the right tempo and key
- Making your profile look complete and professional
- Tracking metadata-complete progress for XP

## What Metadata-Complete Means

BeatPass treats a track as metadata-complete when it has a **description**, **BPM**, **key**, **scale**, and **tags**. Meeting this check awards 50 XP. Good metadata is a quality target, not just a box-ticking exercise.

## Best Practices

<AccordionGroup>
  <Accordion title="Enter accurate musical data">
    BPM, key, and scale should match the actual beat. These fields directly affect search usefulness.
  </Accordion>

  <Accordion title="Use clear naming">
    Clean titles and readable descriptions make the beat feel more professional.
  </Accordion>

  <Accordion title="Tag honestly">
    Use genres and tags that genuinely reflect the beat instead of stuffing unrelated keywords.
  </Accordion>

  <Accordion title="Review older uploads">
    Updating older beats with better cover art, tags, and descriptions can improve the overall quality of your catalog.
  </Accordion>
</AccordionGroup>
