BeatPass requires cover art on new tracks and new collections. The current artwork system supports two main approaches: your own uploaded image, or a genre-matched option from BeatPass's **AI Cover Presets** library.

## Where Cover Art Appears

BeatPass uses track and collection artwork across the product, including:

- Track pages
- Search and browse results
- Producer profiles
- Collection pages

## Current Image Rules

| Requirement | Current Behavior |
|-------------|------------------|
| File types | Image uploads only |
| Max file size | 2 MB |
| Recommended shape | Square |
| Recommended quality | Large enough to still look good after cropping |

<Tip>
  Square images work best because BeatPass presents artwork in a circular display treatment in parts of the UI.
</Tip>

---

## Required on New Saves

The live upload validation currently requires:

- Cover art on new tracks
- Cover art on new collections

Do not treat cover art as optional on brand-new uploads.

## Artwork Priority

For existing content, BeatPass generally prefers:

1. Track-level cover art
2. Collection cover art if the track belongs to one
3. Fallback artwork where applicable

But for new uploads, you should treat cover art as a required part of the save flow.
