The upload page is where producers add new beats and build collections. This guide reflects the current upload flow, field requirements, and validation behavior in BeatPass.

---

## Accessing the Upload Page

To upload as a producer:

1. Sign in to your BeatPass account
2. Open your producer workspace
3. Go to the upload tools

<Note>
  Upload tools are only available to accounts with producer access.
</Note>

---

## Upload Modes

The upload page currently supports two working modes:

| Mode | Best For |
|------|----------|
| **Single-beat upload** | Standalone track releases |
| **Create Collection** | Uploading multiple beats as a collection |

BeatPass uses a toggle labeled **Create Collection**. Turn it on before adding files if you want the upload to build a collection.

<Warning>
  Once files are already in progress, changing modes is not the intended workflow. Reset or finish the current batch before starting a different upload type.
</Warning>

---

## Supported Audio Format

The current producer upload validation only accepts **WAV** files.

<Warning>
  Even though some older UI text mentions other lossless formats, the live upload validation currently blocks non-WAV extensions. Use WAV files if you want the upload to proceed.
</Warning>

## Ways to Add Files

### Drag and Drop

Drag WAV files from your computer into the upload area.

### Choose Files to Upload

Click **Choose Files to Upload** and select your WAV files from the file picker.

---

## What Happens After Upload

After the file is accepted, BeatPass opens the upload form and begins duplicate checking.

The current upload form is organized into these sections:

- **Cover Art & Style**
- **Track Details**
- **Music Properties**
- **Protection & Verification**
- **Licensing Options**

---

## Required Fields for New Tracks

| Field | Required? | Notes |
|-------|-----------|-------|
| **Cover Art** | Yes | Custom upload or AI preset |
| **Track Name** | Yes | Must be a clean track title |
| **Artists** | Yes | At least one credited producer |
| **Key** | Yes | Choose from the current key selector |
| **Scale** | Yes | Major or Minor |
| **BPM** | Yes | Must be between 60 and 200 |
| **Audio Source** | Yes | Created by the upload itself |
| **Licensing Model** | Yes | Standard, Standard + Exclusive, or Exclusive Only |

If the licensing model supports exclusives, BeatPass also requires:

- **Exclusive status**
- **Exclusive contact email**

---

## Cover Art on the Upload Page

BeatPass requires cover art for new tracks before saving.

You can currently use:

- A custom uploaded image
- An **AI Preset** matched from your selected genres

See [Cover Art](/help/uploading/cover-art) for the full artwork rules.

---

## Audio Fingerprint Status During Upload

While uploading, BeatPass shows these current fingerprint states:

| Status | Meaning |
|--------|---------|
| **Pending** | Upload is present but duplicate checking has not started yet |
| **Checking** | BeatPass is starting duplicate detection |
| **Analyzing** | BeatPass is analyzing the audio fingerprint |
| **Verified Unique** | No duplicate was found |
| **Check Failed** | The duplicate check failed, but BeatPass still lets you continue |

<Note>
  A duplicate-check failure does not automatically block the upload. BeatPass currently fails open in that situation, which means you can still continue and save the track.
</Note>

---

## Duplicate Detection

BeatPass checks uploaded audio against existing tracks. If a matching or near-matching beat is detected, BeatPass can block the save flow and show the existing track details.

If the file is unique, the track continues normally.

See [Audio Fingerprinting](/help/uploading/audio-fingerprinting) for the detailed behavior.

---

## Sample-Safe Status

In **Protection & Verification**, BeatPass asks you to choose one of two sample-clearance states:

<CardGroup cols={2}>
  <Card title="Cleared" icon="shield">
    Use this when the beat is 100% original or all samples are fully cleared.
  </Card>
  <Card title="Needs Clearance" icon="warning">
    Use this when the beat contains samples that still require outside clearance.
  </Card>
</CardGroup>

If you choose **Needs Clearance**, BeatPass shows an additional notes field and warns buyers accordingly.

---

## Licensing Options

The current licensing model labels on the form are:

| Option | Meaning |
|--------|---------|
| **Standard Licensing** | Standard BeatPass subscription licensing only |
| **Standard + Exclusive** | Offer both standard licensing and exclusive purchase |
| **Exclusive Only** | Offer the beat only as an exclusive opportunity |

BeatPass automatically shows the extra exclusive fields when you choose an exclusive-capable model.

---

## Saving the Upload

When the required fields are complete:

- Click **Save Track** for single beats
- Use the collection save flow when **Create Collection** is enabled

For collection uploads, remember that:

- The collection itself needs cover art
- Each new track still needs its own audio and cover art

---

## Tips for a Cleaner Upload Workflow

<AccordionGroup>
  <Accordion title="Prepare WAV exports first">
    BeatPass currently blocks non-WAV files, so convert your final masters before you start uploading.
  </Accordion>

  <Accordion title="Pick genres early">
    Genres affect discovery and also drive the AI preset artwork suggestions.
  </Accordion>

  <Accordion title="Do not skip cover art">
    Cover art is required on new tracks and collections, so choose it before you try to save.
  </Accordion>

  <Accordion title="Watch the fingerprint state">
    A beat cannot be confirmed unique until the duplicate check finishes. If BeatPass finds a duplicate, resolve that before expecting the save to succeed.
  </Accordion>
</AccordionGroup>
