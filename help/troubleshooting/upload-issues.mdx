This guide is for approved producers using BeatPass's upload flow.

<Note>
  Standard users cannot upload tracks. If you need producer access first, use the Producer Program.
</Note>

## I Can't Find the Upload Flow

<AccordionGroup>
  <Accordion title="I am not an approved producer">
    Upload access is only available to approved producers.
  </Accordion>
  <Accordion title="I am signed in but still can't upload">
    Open the upload route directly at `/backstage/upload` and confirm the account you are using has producer access.
  </Accordion>
</AccordionGroup>

## The Upload Fails Immediately

<AccordionGroup>
  <Accordion title="Wrong audio format">
    BeatPass currently requires **WAV** audio for new track uploads. Compressed formats like MP3 are not valid for the standard producer upload flow.
  </Accordion>
  <Accordion title="Connection or browser issue">
    Retry on a stable connection, keep the tab open during upload, and test another modern browser if needed.
  </Accordion>
  <Accordion title="The file is unusually large">
    BeatPass can struggle with very large uploads on unstable connections. Retry on a stronger network before assuming the file itself is invalid.
  </Accordion>
</AccordionGroup>

## Save Is Blocked by Required Fields

Before a new track can be saved, BeatPass currently expects:

- Uploaded audio
- Cover art
- BPM between **60** and **200**
- The rest of the required track metadata

For collections, BeatPass also requires:

- Album cover art
- Audio and image data for each new track in the collection

## Duplicate Detection Problems

BeatPass checks uploaded audio for duplicates.

<AccordionGroup>
  <Accordion title="I got a duplicate-audio warning">
    BeatPass believes the new upload matches an existing track. Check whether you already uploaded the track or are trying to replace an older version.
  </Accordion>
  <Accordion title="I think it is a false match">
    Contact support with the current file name, the matched track name if shown, and why you believe the detection is wrong.
  </Accordion>
</AccordionGroup>

## Processing Seems Stuck

After upload, BeatPass may still need to:

- Generate waveform data
- Extract metadata
- Run fingerprint-related checks

If a track looks stuck:

1. Refresh the page
2. Check whether the upload actually completed
3. Retry only after confirming the first attempt did not already save

## Collection Upload Issues

<AccordionGroup>
  <Accordion title="I meant to upload a collection, not singles">
    Use the upload flow's collection mode so BeatPass groups the files as one collection.
  </Accordion>
  <Accordion title="Some files worked and others failed">
    Retry only the failed items after fixing the specific issue for those files.
  </Accordion>
</AccordionGroup>

## Account Limits

BeatPass can show an upload-minutes notice on the upload page. If you run into an account limit, review the message shown in the upload flow before retrying.

## Still Having Issues?

Contact support with:

1. The exact error message
2. The file name and format
3. Whether this was a single-track or collection upload
4. A screenshot if possible
