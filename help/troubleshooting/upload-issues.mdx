This guide is for approved producers using the BeatPass upload flow.

<ProducerOnly />

## I Cannot Find the Upload Flow

<AccordionGroup>
  <Accordion title="I am not an approved producer" icon="lock" defaultOpen>
    Upload access is only available to approved producers. Apply through the [Producer Program](/help/producer-program) first.
  </Accordion>
  <Accordion title="I am signed in but still cannot upload" icon="circle-question">
    Select **Upload beats** in the top navigation bar. If the button is not visible, confirm the account you are using has producer access.
  </Accordion>
</AccordionGroup>

## The Upload Fails Immediately

<AccordionGroup>
  <Accordion title="Wrong audio format" icon="file-audio" defaultOpen>
    BeatPass requires **WAV** audio for new track uploads. If you upload a non-WAV file, an **Unsupported File Format** dialog appears showing the blocked file and the accepted format. Convert to WAV before retrying.
  </Accordion>
  <Accordion title="Connection or browser issue" icon="wifi">
    Retry on a stable connection, keep the tab open during upload, and test another modern browser if needed.
  </Accordion>
  <Accordion title="The file is unusually large" icon="file">
    Large uploads can struggle on unstable connections. Retry on a stronger network before assuming the file itself is invalid.
  </Accordion>
</AccordionGroup>

## Save Is Blocked by Required Fields

Before a new track can be saved, the following fields are required:

| Field | Type | Notes |
| --- | --- | --- |
| **Track Name** | Text | Required |
| **Key** | Select | C through B |
| **Scale** | Select | Major or Minor |
| **BPM** | Number | 50–200 |

You also need an uploaded audio file. Cover art, artists, and licensing options are optional or have auto-defaults.

For collections, BeatPass also requires album cover art and audio for each track in the collection.

## Duplicate Detection

BeatPass checks uploaded audio for duplicates using fingerprint matching.

<AccordionGroup>
  <Accordion title="I got a Duplicate Audio Detected warning" icon="copy" defaultOpen>
    The **Duplicate Audio Detected** dialog shows the original track name, artists, fingerprint date, and a similarity score. Check whether you already uploaded the track or are trying to replace an older version.
  </Accordion>
  <Accordion title="I think it is a false match" icon="circle-question">
    Contact support with the current file name, the matched track name shown in the dialog, and why you believe the detection is incorrect.
  </Accordion>
</AccordionGroup>

## Processing Seems Stuck

After upload, BeatPass may still need to generate waveform data, extract metadata, and run fingerprint checks. If a track looks stuck:

1. Refresh the page
2. Check whether the upload actually completed
3. Retry only after confirming the first attempt did not already save

## Collection Upload Issues

<AccordionGroup>
  <Accordion title="I meant to upload a collection, not singles" icon="layer-group">
    Toggle **Create Collection** in the upload flow so BeatPass groups the files as one collection.
  </Accordion>
  <Accordion title="Some files worked and others failed" icon="circle-xmark">
    Retry only the failed items after fixing the specific issue for those files.
  </Accordion>
</AccordionGroup>

## Account Limits

BeatPass can show an upload-minutes notice on the upload page. If you hit an account limit, review the message shown in the upload flow before retrying.

## Still Having Issues?

Contact support at **contact@beatpass.ca** with:

1. The exact error message or screenshot
2. The file name and format
3. Whether this was a single-track or collection upload
