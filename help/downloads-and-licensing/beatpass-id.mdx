BeatPassID is the fingerprint data BeatPass attaches to tracks for certificate and verification use. It helps BeatPass connect a license to the analyzed audio attached to that track.

<Info>
  For most users, BeatPassID matters because it affects certificate readiness and what appears on the certificate or verification screens.
</Info>

## Where you will see BeatPassID

You may see fingerprint details in:

- License certificates
- The license certificate modal
- Public verification-related certificate data
- Producer-facing upload or track-management flows

## Current fingerprint statuses

| Status | What it means |
| --- | --- |
| **Pending** | BeatPass is still processing the track fingerprint |
| **Completed** | Fingerprint data is ready for certificate tools |
| **Failed** | BeatPass could not complete the fingerprint |

## Why fingerprint status matters

<AccordionGroup>
  <Accordion title="Certificate access" icon="file-certificate" defaultOpen>
    The current **Get License** flow depends on a completed fingerprint. If fingerprinting is still pending, the track may already be downloaded while the certificate view is not ready yet.
  </Accordion>
  <Accordion title="Verification details" icon="shield-check">
    Completed fingerprint data lets BeatPass show the fingerprint section in the certificate and verification-related views.
  </Accordion>
  <Accordion title="Producer workflows" icon="music">
    Producers also encounter fingerprint status during upload and catalog management because BeatPass uses it as part of track-processing workflows.
  </Accordion>
</AccordionGroup>

## What fingerprint details can include

When a completed fingerprint is available, certificate tools can show:

- Fingerprint status
- Fingerprinted date
- Analyzed duration
- Fingerprint hash

## What not to assume

<Warning>
  BeatPassID is useful verification data, but you should not treat it as a guarantee that every certificate is instantly ready or that every rights dispute is solved automatically. The certificate, the license status, and the public verification page still matter.
</Warning>

## Common questions

<AccordionGroup>
  <Accordion title="Why did my track download but I still cannot open the certificate?" icon="download" defaultOpen>
    The download flow can finish before fingerprinting does. When that happens, wait for fingerprint completion and check **Library** -> **Licenses** again.
  </Accordion>
  <Accordion title="What if fingerprint status is failed?" icon="triangle-exclamation">
    Certificate tools may stay unavailable until the track is reprocessed or fixed by BeatPass. The page or workflow where you saw the failure is the best place to start troubleshooting.
  </Accordion>
  <Accordion title="Does BeatPassID change the audio file I download?" icon="wave-square">
    No. It is metadata about the track analysis, not a change to the downloaded file itself.
  </Accordion>
</AccordionGroup>
