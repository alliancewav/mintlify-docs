## Overview

BeatPass is fully accessible on mobile devices through our responsive web application. This guide covers the mobile experience and tips for optimal use.

---

## Accessing on Mobile

### Mobile Web App

BeatPass works in any modern mobile browser:

1. Open your mobile browser (Chrome, Safari, Firefox)
2. Navigate to `open.beatpass.ca`
3. Log in or create an account
4. Start browsing and listening

### Add to Home Screen

For an app-like experience:

<Tabs>
  <Tab title="iOS (Safari)">
    1. Open BeatPass in Safari
    2. Tap the Share button (box with arrow)
    3. Scroll and tap "Add to Home Screen"
    4. Name it and tap "Add"
  </Tab>
  
  <Tab title="Android (Chrome)">
    1. Open BeatPass in Chrome
    2. Tap the menu (three dots)
    3. Select "Add to Home Screen"
    4. Confirm the name and tap "Add"
  </Tab>
</Tabs>

<Tip>
  Adding to home screen enables fullscreen mode and faster loading.
</Tip>

### Mobile Player

The mobile player has been fully restored with a premium **glassmorphism** design:

- **Immersive Overlay**: A beautiful, blurred background that adapts to your cover art.
- **Smart Queue**: Easily see "Now Playing" and "Up Next" tracks with a swipe.
- **Gesture Controls**: Swipe down to minimize, tap to expand.
- **Performance Mode**: Optimized for battery life (video backgrounds disabled on mobile).
- **Play/pause controls**
- **Skip forward/back**
- **Volume control**
- **Like and save buttons**

### Gesture Controls

| Gesture | Action |
|---------|--------|
| Tap waveform | Seek to position |
| Swipe up on player | Expand full player |
| Swipe down | Minimize player |
| Double tap | Like track |

## Mobile Optimization

### Data Usage

Control data consumption:

| Setting | Data Usage |
|---------|------------|
| High Quality | ~2MB per minute |
| Normal | ~1MB per minute |
| Data Saver | ~0.5MB per minute |

Configure at: Settings > Playback > Audio Quality

### Offline Listening

<Warning>
  **BeatPass is streaming-only.** There is no offline listening mode or mobile download feature. Streaming requires an active internet connection.
  
  - Downloaded beats (via subscription on desktop) can be used offline in your DAW/projects
  - Streaming the catalog always requires internet
  - Offline mobile playback is planned for future updates
</Warning>

### Battery Optimization

- Close unused browser tabs
- Use dark mode in low-light conditions
- Disable autoplay when not actively listening
- Lower screen brightness

## Background Playback

### How It Works

BeatPass supports background audio playback on mobile devices:

| Platform | Background Playback | Lock Screen Controls |
|----------|--------------------|--------------------|
| iOS (Safari) | Supported | Play/pause, skip |
| Android (Chrome) | Supported | Play/pause, skip |
| PWA (Home Screen) | Best experience | Full controls |

### Enabling Background Playback

<Tabs>
  <Tab title="iOS">
    1. Audio plays automatically in background when you lock or switch apps
    2. Control playback from lock screen or Control Center
    3. No special settings required
    
    <Note>
      If audio stops when you lock your phone, check that Focus/Do Not Disturb isn't blocking media.
    </Note>
  </Tab>
  
  <Tab title="Android">
    1. Audio continues when you switch apps or lock screen
    2. Control from notification shade or lock screen
    3. Ensure BeatPass isn't restricted by battery optimization
    
    <Tip>
      Add BeatPass to your battery optimization whitelist for uninterrupted playback.
    </Tip>
  </Tab>
</Tabs>

### Lock Screen Controls

| Control | iOS | Android |
|---------|-----|---------|
| Play/Pause | Yes | Yes |
| Skip Forward | Yes | Yes |
| Skip Back | Yes | Yes |
| Seek | No | No |
| Track Info | Yes | Yes |
| Album Art | Yes | Yes |

### Background Playback Limitations

| Scenario | Behavior |
|----------|----------|
| Browser closed completely | Audio stops |
| Phone call received | Audio pauses, resumes after |
| Low memory | OS may stop background audio |
| Data saver mode | May affect streaming quality |

## In-App Browser Limitations

<Warning>
  Opening BeatPass in in-app browsers (Instagram, TikTok, Twitter, Facebook) may cause issues. Use your device's default browser for the best experience.
</Warning>

### Common In-App Browser Issues

| Browser | Known Issues |
|---------|--------------|
| **Instagram WebView** | Login may fail; audio playback unreliable |
| **TikTok WebView** | Limited functionality; session issues |
| **Twitter/X WebView** | Generally works but may have playback issues |
| **Facebook WebView** | Login redirects may fail |
| **Snapchat WebView** | Severely limited functionality |

### How to Open in Default Browser

<Tabs>
  <Tab title="iOS">
    1. Tap the **⋮** or **...** menu in the in-app browser
    2. Select **Open in Safari** or **Open in Browser**
    3. BeatPass will open in Safari with full functionality
  </Tab>
  
  <Tab title="Android">
    1. Tap the **⋮** menu in the in-app browser
    2. Select **Open in Chrome** or **Open in Browser**
    3. BeatPass will open in your default browser
  </Tab>
</Tabs>

### Recommended Approach

When sharing BeatPass links to social media:

1. Users clicking from social apps may land in in-app browser
2. Encourage them to open in their default browser
3. Consider adding "Open in browser for best experience" to descriptions

### Features Affected in WebViews

| Feature | Status in WebViews |
|---------|-------------------|
| Browse & Search | Usually works |
| Playback | May be unreliable |
| Login/Signup | Often fails |
| Purchases | Not recommended |
| File uploads | Usually fails |
| Push notifications | Not supported |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Audio stops when screen locks">
    **iOS**: Audio should continue. If not, check Focus/Do Not Disturb settings.
    **Android**: Ensure BeatPass isn't restricted by battery optimization.
  </Accordion>
  
  <Accordion title="Page not loading correctly">
    - Clear browser cache
    - Try closing and reopening browser
    - Check internet connection
    - Try a different browser
  </Accordion>
  
  <Accordion title="Login issues">
    - Clear cookies for beatpass.ca
    - Disable any VPN or proxy
    - Try "Forgot Password" if needed
  </Accordion>
  
  <Accordion title="Player not appearing">
    - Refresh the page
    - Check if audio is muted system-wide
    - Try force-closing and reopening browser
  </Accordion>
</AccordionGroup>
