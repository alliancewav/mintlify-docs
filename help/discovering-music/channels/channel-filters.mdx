Channel filters are optional. BeatPass only renders them when the channel configuration explicitly enables filters and the channel type supports them.

## When Filters Do Not Show

Filters are hidden when:

- The channel is a nested child inside another channel
- The channel is itself just a container of other channels
- Filtering is disabled for that channel

## Current Filter Mapping

| Content model | Available filters |
| --- | --- |
| Tracks | Search, genre, artist |
| Albums | Search, genre, artist |
| Artists | Search, genre |
| Playlists | Search |
| Users | Search |
| Genres | Search |
| Nested channel containers | None |

## Current Filter Labels

When the channel supports them, BeatPass currently uses:

- `Search...`
- **All Genres**
- **All Artists**

<Note>
  Some channel headers integrate the filters directly into the page header instead of showing them in a separate filter block below it.
</Note>
