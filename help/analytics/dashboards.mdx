## Overview

BeatPass provides comprehensive analytics through the Backstage Insights dashboard. Monitor performance, track earnings, and understand your audience.

<Info>
  **Audience:** Producers
</Info>

---

## Accessing Analytics

### Navigation

1. Log in to BeatPass
2. Go to **Backstage** from the navigation
3. Select your artist profile
4. Click **Insights**

## Reading Your Data

### Plays Chart

The plays timeline shows streaming activity over time:

- **Spikes** indicate viral moments or promotion success
- **Steady line** shows consistent organic engagement
- **Drops** may indicate seasonal patterns or content gaps

### Engagement Breakdown

Understand how listeners interact. We now track **partial listens** (over 10 seconds), giving you a complete picture of engagement even if users skip or leave early.

| Metric              | Good Sign                   | Warning Sign                    |
| ------------------- | --------------------------- | ------------------------------- |
| Completion Rate     | \> 60%                      | \< 30%                          |
| Save Rate           | \> 5%                       | \< 1%                           |
| Repeat Rate         | \> 10%                      | \< 2%                           |
| **Partial Listens** | High count means good hooks | Low count means immediate skips |

### Contribution Trends

Track your earning trajectory:

- Rising trend = Growing engagement value
- Flat trend = Stable but not growing
- Declining = May need content refresh

## Track-Level Analytics

### Individual Track View

Click any track to see detailed analytics:

<Steps>
  <Step title="Select Track">
    From your Backstage Tracks list, click a track.
  </Step>
  <Step title="View Insights">
    Navigate to the Insights tab on the track page.
  </Step>
  <Step title="Analyze">
    Review plays, completions, saves, and trends.
  </Step>
</Steps>

### Track Metrics

| Metric           | Description           |
| ---------------- | --------------------- |
| Total Plays      | Lifetime streams      |
| Unique Listeners | Distinct users        |
| Completion Rate  | % reaching 80%+       |
| Peak Day         | Best single day       |
| Contribution     | Track's earning value |

  <Accordion title="Daily Plays">

### Red Flags

| Indicator                   | Possible Issue                    |
| --------------------------- | --------------------------------- |
| High plays, low completions | Content not engaging              |
| Sudden drop in plays        | Algorithm change or content issue |
| Zero saves                  | Track not resonating              |
| Declining contribution      | Relative performance dropping     |

## Analytics Scope & Availability

### What's Available to Producers

The public dashboards in Backstage Insights provide access to:

| Data Category        | Available | Notes                 |
| -------------------- | --------- | --------------------- |
| Play counts          | Yes       | Total and per-track   |
| Unique listeners     | Yes       | Deduplicated by user  |
| Completion rates     | Yes       | % reaching 80%+       |
| Save counts          | Yes       | Library saves         |
| Contribution value   | Yes       | Estimated earnings    |
| Top tracks           | Yes       | Ranked by performance |
| Date range filtering | Yes       | Up to 24 months       |
| Basic trends         | Yes       | Period-over-period    |

### What's NOT Available (Internal Only)

<Warning>
  The following analytics features are internal and not exposed to end users:
</Warning>

| Feature                     | Reason                             |
| --------------------------- | ---------------------------------- |
| **Real-time analytics**     | Reserved for platform monitoring   |
| **Live listener counts**    | Performance and privacy concerns   |
| **Advanced charts API**     | Internal tooling only              |
| **Data exports (CSV/JSON)** | Not yet available for self-service |
| **Geographic breakdowns**   | Aggregated only, no granular data  |
| **Demographic data**        | Not exposed to producers           |
| **Per-play timestamps**     | Privacy protected                  |
| **Revenue per stream**      | Contribution formula internals     |

### Why Some Data Is Internal

| Reason          | Explanation                                             |
| --------------- | ------------------------------------------------------- |
| **Privacy**     | Listener-level data is protected                        |
| **Performance** | Real-time queries are expensive                         |
| **Fairness**    | Prevents gaming and manipulation                        |
| **Complexity**  | Some metrics are difficult to interpret without context |

### Setting Expectations

- **Public dashboards are the only analytics available** to producers
- Internal analytics exist for platform operations but aren't user-facing
- Export features may be added in future but aren't currently roadmapped
- If you need data we don't expose, contact support to discuss options

<Note>
  We're committed to providing useful analytics while protecting user privacy and platform integrity. If you have feedback on what metrics would help your business, let us know via support.
</Note>
