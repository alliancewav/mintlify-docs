The Finances page is the earnings view inside the Producer Dashboard. Depending on your Stripe status, it either shows onboarding or a reporting dashboard for the current artist.

<Info>
  Open it at <a href="https://open.beatpass.ca/producer-dashboard/finances">https://open.beatpass.ca/producer-dashboard/finances</a>.
</Info>

## Two page states

<CardGroup cols={2}>
  <Card title="Setup Required" icon="triangle-exclamation">
    You see onboarding when Stripe is not fully connected or the artist account is not yet ready to charge and pay out.
  </Card>
  <Card title="Active" icon="wallet">
    You see balances, artist switching, cache timestamps, and transaction history after Stripe setup is complete.
  </Card>
</CardGroup>

## Setup Required

When setup is incomplete, BeatPass shows onboarding instead of incomplete finance data.

### What onboarding covers

- Connecting the artist to Stripe
- Confirming the artist country before starting setup
- Explaining the revenue streams shown in this dashboard

### Revenue streams currently highlighted

- Subscription payouts
- Exclusive sales
- Track purchases

<Note>
  The page only switches into the full dashboard when the connected Stripe account is ready for both charges and payouts.
</Note>

## Active dashboard

Once Stripe is ready, the page becomes a reporting dashboard for the selected artist.

### Header and artist switcher

The top of the page can show:

- The selected artist
- An **Active** or **Setup Required** status badge
- An artist selector when your account manages multiple artists
- Cache timing information when the finance response includes it

## Balance Overview

The balance section uses four cards.

| Card | What it means |
| --- | --- |
| **Balance** | Funds currently available |
| **Upcoming** | Funds that are not yet available |
| **In Transit** | Funds already moving to payout |
| **Lifetime** | Lifetime earned volume shown in the finance response |

### Warnings and notices you may see

- A cache timestamp
- A stale-cache warning
- A multi-currency warning
- A transaction-truncation warning when only recent activity is shown

## Transactions

The lower section combines finance events into one list.

### Filters currently available

- **All**
- **Sales**
- **Subscriptions**
- **Paid**
- **Pending**
- **Failed**

### Status labels currently used

| Status | Meaning |
| --- | --- |
| **Paid** | Completed successfully |
| **Queued** | Waiting in the payout pipeline |
| **Sending** | Transfer is being sent |
| **Pending** | Still processing |
| **Failed** | Did not complete |
| **Excluded** | Not included in the payout distribution |

<Tip>
  Use this page to confirm status and direction of funds. Use the [Earnings docs](/help/earnings) when you need the payout model behind those numbers.
</Tip>

## What this page does not promise

<AccordionGroup>
  <Accordion title="Exact payout timing" icon="clock" defaultOpen>
    The page shows status and balance data, but it does not promise a fixed arrival time for payouts.
  </Accordion>
  <Accordion title="Every Stripe control" icon="credit-card">
    This dashboard is not a full replacement for every Stripe screen or payout-management control.
  </Accordion>
  <Accordion title="Perfectly fresh data on every load" icon="arrows-rotate">
    BeatPass can reuse cached finance responses and explicitly labels them when it does.
  </Accordion>
</AccordionGroup>
