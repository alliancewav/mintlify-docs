## Overview

This catalog documents common errors you may encounter across BeatPass API endpoints, along with their meanings and suggested resolutions.

---

## Error Response Format

All errors follow this structure:

```json
{
  "message": "Human-readable error description",
  "errors": {
    "field_name": ["Specific validation error"]
  }
}
```

---

## HTTP Status Codes

| Code | Name | General Meaning |
|------|------|-----------------|
| 400 | Bad Request | Malformed request syntax |
| 401 | Unauthorized | Authentication required or invalid |
| 403 | Forbidden | Authenticated but not permitted |
| 404 | Not Found | Resource doesn't exist |
| 409 | Conflict | Resource state conflict |
| 422 | Unprocessable Entity | Validation failed |
| 429 | Too Many Requests | Rate limit exceeded |
| 500 | Internal Server Error | Server-side error |

## Authentication Errors

### Login & Session

| Error | Code | Cause | Resolution |
|-------|------|-------|------------|
| `Invalid credentials` | 401 | Wrong email/password | Verify credentials |
| `Account disabled` | 403 | Account suspended | Contact support |
| `Email not verified` | 403 | Email verification pending | Check inbox for verification link |
| `Session expired` | 401 | Token/session timed out | Re-authenticate |
| `CSRF token mismatch` | 419 | Missing/invalid CSRF token | Refresh page, get new token |

### Session Errors

| Error | Code | Cause | Resolution |
|-------|------|-------|------------|
| `Unauthenticated` | 401 | Missing session cookie | Authenticate via login |
| `Session expired` | 401 | Session no longer valid | Re-authenticate |
| `Invalid session` | 401 | Malformed or revoked session | Re-authenticate |

## Track & Upload Errors

### Upload Validation

| Error | Code | Cause | Resolution |
|-------|------|-------|------------|
| `File too large` | 422 | Exceeds max file size | Reduce file size (max ~250MB) |
| `Invalid file format` | 422 | Unsupported audio format | Use WAV, FLAC, or AIFF (MP3 not accepted) |
| `Duration too short` | 422 | Under 30 seconds | Upload longer track |
| `Duration too long` | 422 | Over 15 minutes | Trim to under 15 minutes |
| `Title is required` | 422 | Missing track title | Provide title |
| `BPM is required` | 422 | Missing BPM | Provide BPM value |
| `Genre is required` | 422 | Missing genre | Select genre |

### Processing Errors

| Error | Code | Cause | Resolution |
|-------|------|-------|------------|
| `Transcode failed` | 500 | Audio processing error | Re-export and re-upload |
| `Fingerprint collision` | 409 | Duplicate detected | Track may already exist |
| `Processing timeout` | 504 | File took too long | Try smaller file or retry later |

### Track Access

| Error | Code | Cause | Resolution |
|-------|------|-------|------------|
| `Track not found` | 404 | ID doesn't exist or deleted | Verify track ID |
| `Track unavailable` | 410 | Removed by owner/moderation | Track no longer accessible |
| `Private track` | 403 | Track not published | Only owner can access |

## Beat Request Errors

### Creation

| Error | Code | Cause | Resolution |
|-------|------|-------|------------|
| `Request limit reached` | 429 | Max active requests | Close old requests first |
| `Title is required` | 422 | Missing title | Provide title |
| `Description too short` | 422 | Under minimum length | Add more detail |

### Response

| Error | Code | Cause | Resolution |
|-------|------|-------|------------|
| `Request not found` | 404 | Invalid request ID | Verify ID |
| `Request closed` | 410 | Request no longer accepting responses | Find open request |
| `Already responded` | 409 | User already submitted response | Cannot submit twice |
| `Request expired` | 410 | Past 30-day limit | Request auto-expired |

### Status Changes

| Error | Code | Cause | Resolution |
|-------|------|-------|------------|
| `Cannot close` | 403 | Not request owner | Only owner can close |
| `Already fulfilled` | 409 | Request already completed | No action needed |

| Error | Code | Cause | Resolution |

| `Playlist full` | 429 | Max tracks per playlist (500) | Create new playlist |
| `Playlist limit reached` | 429 | Max playlists (100) | Delete unused playlists |
| `Track already saved` | 409 | Already in library | No action needed |
| `Track already in playlist` | 409 | Duplicate in playlist | Track exists |

### Collaborative Playlists

## Rate Limiting

### Rate Limit Response

```json
{
  "message": "Too Many Requests",
  "retry_after": 60
}
```

### Rate Limit Headers

```http
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1700000060
Retry-After: 60
```

### Handling Rate Limits

| Strategy | Implementation |
|----------|----------------|
| Exponential backoff | Wait, double delay on retry |
| Check headers | Monitor remaining requests |
| Queue requests | Space out API calls |
| Cache responses | Reduce redundant calls |

## Server Errors (5xx)

| Error | Code | Cause | Resolution |
|-------|------|-------|------------|
| `Internal Server Error` | 500 | Unexpected server issue | Retry; report if persistent |
| `Service Unavailable` | 503 | Platform maintenance | Check status page |
| `Gateway Timeout` | 504 | Request took too long | Retry with smaller payload |

<Note>
  500 errors are logged automatically. If you encounter persistent 500 errors, contact support with the approximate time and endpoint.
</Note>

## Troubleshooting Checklist

<Steps>
  <Step title="Check Error Message">
    Read the `message` field for human-readable explanation.
  </Step>
  
  <Step title="Check Validation Errors">
    Review `errors` object for field-specific issues.
  </Step>
  
  <Step title="Verify Authentication">
    Ensure session is valid and cookies are included.
  </Step>
  
  <Step title="Check Rate Limits">
    Review `X-RateLimit-*` headers.
  </Step>
  
  <Step title="Retry with Backoff">
    For transient errors, wait and retry.
  </Step>
  
  <Step title="Contact Support">
    For persistent issues, provide error details and timestamps.
  </Step>
</Steps>
